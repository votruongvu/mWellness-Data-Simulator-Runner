## The only build-fix applied — `Gemfile` (Ruby 3.4 toolchain compat; NO native logic)
The RN 0.74 template pinned CocoaPods `< 1.15` (resolves to 1.14.3), which is
incompatible with this machine's **Ruby 3.4.7**:
1. **`cannot load such file -- kconv`** — Ruby 3.4 moved `nkf` (which provides the
   `kconv` library) out of the default gems. CocoaPods → `xcodeproj` → the
   `CFPropertyList` gem still `require`s `kconv` when reading an XCFramework
   `Info.plist`. Fix: declare **`gem 'nkf'`** so `require 'kconv'` resolves.
2. CocoaPods **`>= 1.16`** (Ruby-3.4-compatible; also fixes the template's noted
   1.15 build bug); `activesupport` upper bound relaxed so bundler resolves the set.

Locked result (`Gemfile.lock`): cocoapods 1.16.2 · xcodeproj 1.27.0 · nkf 0.3.0 ·
activesupport 7.0.10. **No `ios/` or `android/` source was modified.**
